# LIB_Footprint.create

## 这个 API 是干什么的

创建封装

## 什么时候该用它

当你想把 create 放进自动化脚本、批处理或测试脚本时，可以先照着这个案例跑一遍，确认创建后的返回值和清理方式。

## 运行前需要什么上下文

- 是否需要已打开工程：不强制
- 是否需要活动文档：不强制
- 自动验证模式：manual
- 清理策略：auto

## 完整可运行代码

```javascript
return await (async function(eda) {
  const fixtureName = '__codex_example___LIB_Footprint_create_' + Date.now();
  const fixtureSlug = fixtureName.toLowerCase().replace(/[^a-z0-9]+/g, '-');
  const footprintCandidates = await eda.lib_Footprint.search('STM32');
  const footprintItem = footprintCandidates?.[0];
  const footprintUuid = footprintItem?.uuid;
  const footprintLibraryUuid = footprintItem?.libraryUuid;
  const created = await eda.lib_Footprint.create(footprintLibraryUuid, 'footprintName', undefined, 'Generated by Codex example verification');
  const primitiveId = created?.getState_PrimitiveId?.() ?? created?.primitiveId ?? created?.id;
  if (primitiveId) {
    await eda.lib_Footprint.delete([primitiveId]);
  }
  return {
    fixtureName,
    created,
    cleanupStrategy: 'auto',
  };
})(eda);
```

## 参数与返回值怎么理解

- `libraryUuid`：库 UUID，可以使用 LIB\_LibrariesList 内的接口获取
- `footprintName`：封装名称
- `classification`：_（可选）_ 分类
- `description`：_（可选）_ 描述

- 返回值：Promise<string \| undefined>

封装 UUID
